# AI Helpers

The `cachebox_client::ai` module provides provider-neutral helper utilities for AI
workloads. These helpers create stable cache keys, guide lease-based generation
flows, and support conservative streaming capture. They do not call model APIs.

## Prompt And Result Keys

Use `prompt_cache_key` to build a deterministic ASCII byte key for a structured
prompt request:

```rust
use cachebox_client::ai::{PromptCacheKeyInput, PromptMessage, prompt_cache_key};
use serde_json::json;

let mut input = PromptCacheKeyInput::new("openai", "gpt-example", "workspace-a");
input.model_version = Some("2026-06-01".to_string());
input.system_prompt = Some("Answer with citations.".to_string());
input.messages = vec![
    PromptMessage::new("user", "Summarize the release notes."),
    PromptMessage::new("assistant", "Which product?").with_name("cachebot"),
    PromptMessage::new("user", "Cachebox."),
];
input.tool_schema = Some(json!({
    "type": "object",
    "properties": {
        "query": { "type": "string" },
        "limit": { "type": "integer" }
    }
}));
input.sampling_parameters.insert("temperature".to_string(), json!(0.2));
input.output_format = Some("json".to_string());
input.retrieval_context_hash = Some("sha256:abc123".to_string());

let key = prompt_cache_key(&input);
assert!(key.starts_with(b"ai:prompt:v1:"));
```

Normalization rules:

- The payload starts with `cachebox.ai.prompt.v1`.
- Fields are length-prefixed UTF-8 bytes.
- Message order is significant.
- JSON objects are canonicalized with sorted keys and compact separators.
- Sampling parameter names are sorted.
- Missing optional fields and empty strings are distinct.

## Embedding Keys

Use `embedding_cache_key` for embedding generation cache entries:

```rust
use cachebox_client::ai::{EmbeddingCacheKeyInput, embedding_cache_key};
use serde_json::json;

let mut input = EmbeddingCacheKeyInput::new(
    "text-embedding-example",
    "sha256:contentabc",
    "markdown:v1:512:64",
    1536,
    "workspace-a",
);
input.model_version = Some("2026-06-01".to_string());
input.normalization_settings.insert("unicode".to_string(), json!("nfc"));
input.normalization_settings.insert("case_fold".to_string(), json!(false));

let key = embedding_cache_key(&input);
assert!(key.starts_with(b"ai:embedding:v1:"));
```

Embedding keys include:

- Model.
- Optional model version.
- Input content hash.
- Normalization settings.
- Chunking strategy.
- Dimension count.
- Application namespace.

Cachebox stores embeddings as raw bytes. Similarity search belongs in a vector
database or search system.

## Generation Lease Decisions

Use `generation_lease_action` to map lease start results into a client decision:

```rust
use cachebox_client::ai::{
    GenerationLeaseAction, GenerationLeaseStart, generation_lease_action,
};

let action = generation_lease_action(GenerationLeaseStart::LeaseGranted {
    lease_token: "lease-1".to_string(),
    stale_value: None,
});

match action {
    GenerationLeaseAction::ReturnCached(bytes) => {
        // Return the fresh or stale cached bytes.
        let _ = bytes;
    }
    GenerationLeaseAction::Generate { lease_token, stale_value } => {
        // Generate once, then complete the Cachebox lease with fresh bytes.
        let _ = (lease_token, stale_value);
    }
    GenerationLeaseAction::RetryLater => {
        // Another client owns the active lease.
    }
}
```

Completion failures stay explicit:

```rust
use cachebox_client::ai::{
    GenerationCompletionFailure, generation_completion_failure,
};

let failure = generation_completion_failure(
    "lease-1",
    GenerationCompletionFailure::ExpiredLease,
);
```

Clients should only publish generated bytes after lease completion succeeds.

## Streaming Capture

`StreamCapture` implements the conservative buffer-then-commit path for streamed
generation:

```rust
use cachebox_client::ai::StreamCapture;

let mut capture = StreamCapture::new();
capture.push_chunk(b"hello ".to_vec());
capture.push_chunk(b"world".to_vec());

let captured = capture.finish("lease-1").expect("generation succeeded");
assert_eq!(captured.value, b"hello world");
```

Failed generations do not produce a value:

```rust
use cachebox_client::ai::{StreamCapture, StreamCaptureError};

let mut capture = StreamCapture::new();
capture.push_chunk(b"partial".to_vec());
capture.mark_failed();

assert_eq!(
    capture.finish("lease-1"),
    Err(StreamCaptureError::GenerationFailed)
);
```

Cachebox stores the finished stream as one raw-byte value. There is no server-side
chunk append protocol.

## Cost Metadata For AI Workloads

Use native metadata to record a recomputation score:

```rust
use cachebox::protocol::{Metadata, Ttl};

let metadata = Metadata {
    cost: Some(1800),
    ttl: Some(Ttl {
        milliseconds: 600_000,
    }),
    stale_ttl: Some(Ttl {
        milliseconds: 3_600_000,
    }),
    tags: vec![
        "model:gpt-example".to_string(),
        "prompt-template:v3".to_string(),
    ],
    ..Metadata::default()
};

client
    .put(
        "default",
        b"ai:answer".to_vec(),
        metadata,
        b"model output bytes".to_vec(),
    )
    .await?;
```

The currently accounted aggregate cost score is available in Prometheus
metrics:

```text
cachebox_cost_score_total 1800
```

The current eviction policy is approximate LRU. Cost score is stored and
measured for future policy experiments.
